Fase 2 — Integridad
Missingness — Gestión de Datos Faltantes
El mecanismo de ausencia determina el tratamiento. Imputar sin diagnosticar es el error más común y el más dañino. En datos de seguridad crítica, el patrón de ausencia puede ser la señal más valiosa del dataset.
ITRB GISTM ISO 25012
⛔
ITRB/GISTM — Instrucción Crítica: En datos de seguridad de presas (piezómetros, inclinómetros, aforadores de filtración, extensómetros), los valores MNAR NUNCA se imputan con estadísticos simples (media, mediana). El patrón sistemático de ausencia ES la señal de alerta. Conservar + crear indicador binario + escalar al ingeniero responsable.
Clasificación de Mecanismos — Diagnóstico Obligatorio
MCAR
Completamente al Azar
La ausencia no depende de ninguna variable observable ni del valor faltante mismo.
↓Escenario ideal: no introduce sesgo sistémico
↓Verificar con prueba Little's MCAR (scipy/statsmodels)
↓Si % < 5%: eliminación de filas válida
↓Si 5-15%: imputación simple justificada
↓Si >15%: revisar calidad del sensor/proceso
MAR
Al Azar Condicionado
La ausencia depende de variables observables en el dataset.
↓Verificar con regresión logística: ¿predice la máscara de missingness?
↓Imputación condicionada es válida si el modelo de imputación es correcto
↓Series temporales cortas: interpolación lineal o spline cúbico
↓MAR multivariado complejo: MICE (Multiple Imputation by Chained Equations)
↓MAR con predictores fuertes: KNN (k=5, Robust Scaler previo)
MNAR
No Aleatorio — RIESGO ALTO
La ausencia depende del propio valor faltante. El más peligroso.
⚠No existe prueba estadística definitiva para MNAR — requiere análisis de dominio
⚠Crear indicador binario: flag_missing_sensor_X = 1
⚠Documentar contexto operacional del período de ausencia
⛔En datos de seguridad: ESCALAR INMEDIATAMENTE al ingeniero responsable
⛔Nunca imputar sin aprobación del responsable del monitoreo

from sklearn.ensemble import IsolationForest
from scipy.spatial.distance import mahalanobis

# ─────────────────────────────────────────────────────────────
# MÉTODO 1: Modified Z-Score (MAD) — PREFERIDO para hidrología
# Ref: Iglewicz & Hoaglin (1993) — ASTM E178
# MAD es resistente a la influencia de los propios outliers
# ─────────────────────────────────────────────────────────────

def detectar_outliers_mad(serie: pd.Series, threshold=3.5) -> pd.Series:
    """Modified Z-Score — no requiere normalidad, robusto para caudales."""
    mad = median_abs_deviation(serie.dropna(), scale='normal')
    if mad == 0: return pd.Series(False, index=serie.index)
    modified_z = 0.6745 * (serie - serie.median()) / mad
    return abs(modified_z) > threshold

# Aplicar a variable crítica
mask_mad = detectar_outliers_mad(df_v01_raw['caudal_filtracion_ls'])
df_v01_raw['flag_outlier_mad'] = mask_mad.astype(int)

import re

# ─────────────────────────────────────────────────────────────
# REPORTE CALIDAD ISO 25012 — Entregable auditable
# Ref: ISO 25012:2008 — Data Quality Model
# ─────────────────────────────────────────────────────────────

def score_completitud(df) -> dict:
    pct_present = (1 - df.isna().mean()) * 100
    return {'score_global': pct_present.mean().round(2), 'por_variable': pct_present.to_dict()}

def score_unicidad(df) -> dict:
    n_dup = df.duplicated().sum()
    return {'score': round((1 - n_dup/len(df))*100, 2), 'duplicados': int(n_dup)}

def score_validez(df, reglas: dict) -> dict:
    """
    reglas = {'presion_piezometrica': (0, 120),  # msnm físicamente posible
               'caudal_filtracion_ls': (0, 500)}
    """
    scores = {}
    for col, (vmin, vmax) in reglas.items():
        if col in df.columns:
            validos = df[col].between(vmin, vmax, inclusive='both').sum()
            scores[col] = round(validos/df[col].count()*100, 2)
    return scores

# Reglas de negocio — definidas con el equipo de ingeniería
REGLAS_VALIDEZ = {
    'presion_piezometrica': (60.0, 120.0),   # msnm — cota mínima cimentación/coronamiento

    'temperatura_agua_c':   (4.0, 35.0),      # °C — rango físico embalse tropical
}

# Consolidar reporte
reporte_calidad = {

Feature Engineering — Variables que Informan
El 60% del tiempo de un Data Scientist ocurre aquí. No se crean features por intuición: cada variable nueva debe tener justificación estadística y de dominio, y no puede haberse calculado usando datos del set de test.
CRISP-DM §4 WMO SPI
⛔
DATA LEAKAGE — Error fatal: Toda transformación (imputación, escalado, encoding, cálculo de medias) debe ser fit() SOLO en el set de entrenamiento y luego aplicada con transform() a ambos sets. Jamás al revés. Un modelo con leakage funcionará en desarrollo y fallará en producción sin diagnóstico obvio.
Transformaciones No Lineales — Selección Justificada
LOG
Transformación Logarítmica
Cuándo: Distribuciones log-normales confirmadas (caudales, precipitación, sedimentos).
→np.log1p(x) — preferir sobre log(x) para manejar ceros sin offset arbitrario
→Verificar: skewness post-transformación debe ser |γ1| < 0.5
⚠No aplicar si la variable tiene interpretación en escala original para el negocio
BC
Box-Cox / Yeo-Johnson
Cuándo: Sesgo fuerte, λ óptimo desconocido. Automático y auditable.
→Box-Cox: solo x > 0. scipy.stats.boxcox() optimiza λ
→Yeo-Johnson: permite x ≤ 0 (diferencias de nivel, anomalías). Preferir en instrumentación
→Documentar el λ estimado: es un parámetro aprendido en train
SCL
Escalado — Selección Crítica
EscaladoCuándo
Robust ScalerSeries hídrica con colas pesadas ✓
Z-Score Std.Distribución normal confirmada
Min-MaxRango fijo SIN outliers extremos
MaxAbsVariables con positivos y negativos

def crear_features_hidrologicos(df: pd.DataFrame, col_q: str, col_p: str) -> pd.DataFrame:
    """
    Genera features estándar para modelado hidrológico.
    IMPORTANTE: Llamar solo sobre el set correcto (no contaminar).
    """
    df = df.copy()

    # Rezagos temporales
    for lag in [1, 3, 7, 14, 30]:
        df[f'{col_q}_lag{lag}d'] = df[col_q].shift(lag)

    # Medias móviles — diferente ventana para capturar escala
    df[f'{col_q}_sma7']  = df[col_q].rolling(7, min_periods=4).mean()
    df[f'{col_q}_sma30'] = df[col_q].rolling(30, min_periods=15).mean()

    # Anomalía sobre media móvil (señal de alerta de filtración)
    df[f'{col_q}_anom_pct'] = (df[col_q] / df[f'{col_q}_sma30'] - 1) * 100

    # Componentes estacionales — sin data leakage, solo índices de fecha
    df['mes']          = df.index.month
    df['semana_anio']  = df.index.isocalendar().week.astype(int)
    df['quincena']    = (df.index.day > 15).astype(int)

    # Coeficiente de escorrentía diario (requiere área de cuenca — dominio experto)
    AREA_CUENCA_KM2 = 1250
    df['coef_escorrentia'] = df[col_q] / (df[col_p] * AREA_CUENCA_KM2 * 1000 / 86400 + 1e-6)

    return df

df_v04_train = crear_features_hidrologicos(df_v04_train, 'caudal_m3s', 'precip_mm')

# Pipeline Scikit-Learn — fit en train, transform en ambos
pipe_prepro = Pipeline([
    ('yeo', PowerTransformer(method='yeo-johnson')),  # permite x ≤ 0
    ('scaler', RobustScaler())                        # robusto a outliers residuales
])

cols_numericas = df_v04_train.select_dtypes('number').columns.tolist()
df_v04_train[cols_numericas] = pipe_prepro.fit_transform(df_v04_train[cols_numericas])
df_v04_test[cols_numericas]  = pipe_prepro.transform(df_v04_test[cols_numericas])

Anti-Patrones — Prohibidos
Lo que No Se Hace — Nunca
Estos errores no son opinables. Son fallas metodológicas con impacto directo en la validez del análisis, la reproducibilidad del trabajo y la seguridad de las conclusiones. En contextos de infraestructura crítica, algunos pueden implicar consecuencias legales y de seguridad humana.
TODOS
❌
Imputar sin diagnosticar el mecanismo — MCAR, MAR y MNAR requieren estrategias completamente distintas. Aplicar media a un MNAR en piezómetros es un error de seguridad.
❌
Eliminar outliers sin verificar su naturaleza — Un outlier en filtración de presa puede ser la señal de inicio de falla. Eliminar por rutina sin análisis es inaceptable.
❌
Z-Score en distribuciones sesgadas — Caudales y precipitación son log-normales. Z-Score producirá falsos negativos en la cola derecha (los eventos extremos que más importan).
❌
Min-Max Scaling con outliers extremos — Un único valor extremo comprime toda la distribución al rango [0.001, 0.01]. Usar RobustScaler en series hidrológicas.
❌
Asumir normalidad sin prueba estadística — La normalidad debe ser verificada con Shapiro-Wilk (n<5000) o Anderson-Darling. No se asume nunca. Afecta la elección de correlación, pruebas de hipótesis y modelos.
❌
Data Leakage por imputación antes del split — Calcular la media/mediana con todos los datos antes de separar train/test filtra información del futuro al modelo. El split va primero, siempre.
❌
Avanzar al nivel N+1 de Gartner sin completar N — Construir modelos predictivos sin entender descriptivamente los datos produce modelos que no se pueden diagnosticar cuando fallan.
❌
Resultados no reproducibles — Sin semilla aleatoria fija, sin log de transformaciones, sin hash del dataset, el análisis no puede ser auditado. Inválido para ITRB, ISO 9001 e ICOLD.
❌
Pearson en datos con correlación no lineal — Pearson solo mide correlación lineal. En relaciones curvilíneas (caudal-turbidez, lluvia-escorrentía), Spearman o Kendall son los métodos correctos.
❌
Aplicar técnicas por costumbre sin justificación — "Siempre uso Random Forest" o "Siempre imputo con la media" no son decisiones técnicas. Cada elección debe tener fundamento estadístico explícito.
